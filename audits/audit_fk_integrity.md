## Method

1. Open `auth.db`; pull every row from `sqlite_master` for `type='table'`.
2. For each `sql` blob, regex-match `REFERENCES <ident>(`.
3. Do the same for every `CREATE TABLE` body in `db.py` and each migration,
   parsed with a balanced-paren slice to capture the full column list.
4. Compare each captured target to (a) the set of live tables, and
   (b) the union of every table CREATE-ed anywhere in the schema source.
5. SQLite identifiers are case-insensitive, so the comparison normalises.

Caveats:
- Table-level only. Column-level FK checks (does `target(col)` exist?)
  are out of scope per the brief.
- Migrations that build SQL via string formatting in Python helpers
  (rather than literal `CREATE TABLE` strings) are not parsed. The
  REFERENCES regex still catches any FK clause embedded in helper
  output as long as the literal text survives in the source.
