Stability & versioning

What you can rely on, and what may move. Canonical reference: API_STABILITY.md at the repo root.

Path conventions

  • /api/v1/* — stable. Backward-compatible changes only.
  • /api/internal/* — internal use; may change without notice.
  • /api/* (no version) — deprecated for new consumers; will sunset after v1 fully covers it.

Breaking change policy

  • 6-month deprecation notice via /changelog + email to API key owners.
  • Sunset HTTP header on deprecated endpoints (per RFC 8594).
  • Old version stays live for 12 months after deprecation announcement.

SLA

  • 99.5% uptime target.
  • Response time: p99 < 500 ms for read endpoints.
  • Bulk endpoints rate-limited per the table below.

Authentication

narve.ai uses five auth modes. Pick the one that matches the endpoint group.

Anonymous

Public

No credentials needed. Cacheable. Used by feed, markets, source profiles, changelog, health.

Session cookie

User

Browser-only. Cookie narve_session set by /auth/login. Mutating verbs also require a CSRF pair.

Cookie: narve_session=<opaque>
x-csrf-token: <value of _csrf cookie>

Bearer key

Pro / Enterprise

For server-to-server access on /api/public/v1/*. Mint at /settings/api-keys. Scopes: read, write.

Authorization: Bearer narve_sk_...your-key...

Embed key

Embed

Per-widget key for embeddable JSON endpoints. Origin-locked at issuance.

X-API-Key: nve_pub_...your-key...

Subdomain HMAC SSO

Subproducts

Subproduct APIs (whale, voters, climate, …) live on per-product subdomains. The gateway proxies authenticated requests and injects X-Gateway-Secret (HMAC-derived shared secret) so the upstream can trust the identity headers. Direct subdomain access without going through narve.ai is rejected.

CSRF on mutating verbs

Any POST/PUT/PATCH/DELETE issued with a session cookie must echo the CSRF cookie back as a header:

curl -X POST https://narve.ai/api/saved \
  -H "Cookie: narve_session=$SESSION; _csrf=$CSRF" \
  -H "x-csrf-token: $CSRF" \
  -H "Content-Type: application/json" \
  -d '{"prediction_id": 19281}'

Secret handling

  • Bearer keys are shown once at creation; we store only the SHA-256 hash. Rotate via revoke + mint.
  • Embed keys are origin-locked and rate-limited per widget. Rotate via POST /api/embeds/{id}/rotate-token.
  • Never commit keys to source control. Treat sk_… values the way you'd treat a database password.

Rate limits

Limits are per-key or per-IP and reset on a rolling clock window. 429 responses carry Retry-After seconds plus X-RateLimit-* headers.

Public endpoints

No authentication. Safe to call from a browser or a crawler.

GET /api/feed no auth

Recent extracted predictions across every tracked source. Newest first.

Query: limit (max 500), category, since (unix seconds).

curl https://narve.ai/api/feed?limit=20
{
  "feed": [
    {
      "prediction_id": 19281,
      "source_handle": "nate_silver",
      "market_slug": "us-election-2028",
      "direction": "YES",
      "predicted_probability": 0.62,
      "extracted_at": 1714928400
    }
  ],
  "limit": 20
}
GET /api/markets no auth

Search live Polymarket + Kalshi markets. Returns the latest snapshot per market.

Query: q, limit (max 200), category.

curl "https://narve.ai/api/markets?q=election&limit=10"
{
  "markets": [
    {
      "slug": "us-election-2028",
      "title": "Who wins the 2028 US presidential election?",
      "price_yes": 0.41,
      "venue": "polymarket",
      "narve_probability": 0.48,
      "edge": 0.07
    }
  ],
  "limit": 10
}
GET /api/sources/{handle} no auth

Full credibility profile for one source: calibration, accuracy, category breakdown, scorecard.

curl https://narve.ai/api/sources/nate_silver
{
  "source_handle": "nate_silver",
  "global_credibility": 0.68,
  "total_predictions": 412,
  "correct_predictions": 281,
  "categories": {"politics": 0.74, "sports": 0.61},
  "last_computed_at": 1714928400
}
GET /api/predictions/{id} no auth

One prediction by ID. Includes provenance link, market snapshot at extraction, and current resolution state.

curl https://narve.ai/api/predictions/19281
GET /api/changelog no auth

Structured changelog entries powering /changelog and the in-app widget.

curl https://narve.ai/api/changelog?limit=10

User-scoped endpoints

Session cookie required. Mutating verbs additionally require the CSRF header pair.

GET /api/me/profile session

The signed-in user's profile: handle, email, plan, add-ons, created date, notification preferences.

curl https://narve.ai/api/me/profile \
  -H "Cookie: narve_session=$SESSION"
{
  "user_id": 4711,
  "email": "you@example.com",
  "username": "you",
  "plan": "pro_monthly",
  "addons": ["trading", "voters"],
  "created_at": 1700000000
}
GET /api/me/positions session

All open and closed positions tied to your connected Polymarket + Kalshi accounts. Newest first.

curl https://narve.ai/api/me/positions?status=open \
  -H "Cookie: narve_session=$SESSION"
GET /api/me/bankroll session

Current bankroll, target sizing parameters, and historical PnL. Update with PATCH.

POST /api/me/export session + csrf

Request a GDPR-style data export. Runs as a background job; you receive an email with a signed download link when ready.

curl -X POST https://narve.ai/api/me/export \
  -H "Cookie: narve_session=$SESSION; _csrf=$CSRF" \
  -H "x-csrf-token: $CSRF"

Subscribed endpoints

Pro plan or matching add-on required. Returns 402 if your plan doesn't include the feature.

GET /api/signal-search session · Pro

Full-text search across every tracked prediction. Saved-search topics can be persisted via /api/topics.

Query: q (required), limit, since, categories.

curl "https://narve.ai/api/signal-search?q=bitcoin+halving" \
  -H "Cookie: narve_session=$SESSION"
POST /api/kelly/calculate session · Trading add-on

Compute Kelly-criterion stake size for a market given your bankroll and an estimated edge. Returns full, half, and quarter-Kelly recommendations.

curl -X POST https://narve.ai/api/kelly/calculate \
  -H "Cookie: narve_session=$SESSION; _csrf=$CSRF" \
  -H "x-csrf-token: $CSRF" \
  -H "Content-Type: application/json" \
  -d '{"probability":0.62,"market_price":0.41,"bankroll":10000}'
{
  "full_kelly_pct": 0.355,
  "full_kelly_stake": 3550,
  "half_kelly_stake": 1775,
  "quarter_kelly_stake": 887,
  "edge": 0.21
}
GET /api/markets/portfolio session · Trading add-on

Unified portfolio view across all connected venues. Aggregated by market, by venue, and over time.

Embed widgets

Per-widget API key. Origin-locked to the domains you registered at /settings/embeds.

GET /api/embed/best-bets embed key

Top edge opportunities for the widget. Cached at the edge for 60 seconds.

curl https://narve.ai/api/embed/best-bets \
  -H "X-API-Key: nve_pub_...your-key..."
GET /api/embed/markets embed key

Filtered market list for the widget's configured scope (category, venue, limit).

curl https://narve.ai/api/embed/markets \
  -H "X-API-Key: nve_pub_...your-key..."

Subproduct endpoints

Routed through subdomain SSO. Each call must originate from narve.ai — the gateway injects an HMAC-derived X-Gateway-Secret header that the upstream dashboard validates. Subscription to the specific subproduct (or a Pro plan that bundles it) is required.

GET /api/whale/whales session · whale

Institutional 13F filings cross-referenced against live Polymarket positions. Whale Watch surface.

GET /api/voters/elections session · voters

National polling, V-Dem democracy indicators, election markets per country. Voters Atlas surface.

GET /api/climate/co2 session · climate

Live CO2 (NOAA), GISTEMP, ENSO, sea-ice and methane indicators. Climate Change subproduct.

Other subproducts (Sports, Weather, World, Crypto, Midterm, Traders, Eco Disasters, Central Bank, World Health, …) follow the same pattern — /api/{slug}/… on the same auth + HMAC SSO chain. See /dashboards for the full subproduct catalogue.

Error codes

StatusMeaningWhat to do
400Bad request — invalid query or bodyInspect error.detail and fix the payload.
401Missing / invalid credentialCheck session cookie, Bearer key, or X-API-Key.
402Plan / add-on doesn't include this featureUpgrade at /pricing.
403Key lacks scope, CSRF mismatch, or origin mismatchRe-mint with the right scope, or send the CSRF header pair.
404Resource not foundConfirm the slug / handle / ID.
409Conflict (duplicate, stale state)Refetch and retry with the latest state.
429Rate limit exceededHonour Retry-After. Shard across keys if you have a higher tier.
500Server errorRetry with backoff. Persistent? hello@narve.ai.

Machine-readable schema

The full OpenAPI 3 schema is published at /api/openapi.json. Pass it to SDK generators, agent tooling, or Postman.

curl https://narve.ai/api/openapi.json | jq .info

Internal and admin routes set include_in_schema=False and do not appear in the published schema.

Questions? Email hello@narve.ai — API-tagged messages get a response within two business days.
