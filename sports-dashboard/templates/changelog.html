Pro-tier complete latest

Sharpe now has the full sharp-bettor toolkit. Some highlights:

Verifiable proof of edge

  • Track record — CLV, simulated P&L, calibration plots across every flagged signal. Anonymous-readable.
  • Leaderboard — opt-in roster of users ranked by mean closing line value.

Cross-venue signals

  • Player props — books × Kalshi × Polymarket on (player, stat, line).
  • Cross-book arbitrage — low-hold + middles across major sportsbooks.
  • Smart money — top-50 Polymarket whales' positions overlaid on sports markets.
  • Polymarket fills tape — live large-fill feed via WS.
  • Steam moves — sharp-book line moves ≥2pp within 30 min.

Your edge

  • Bet tracker — per-trade CLV, per-sport/book stats, CSV export.
  • Backtest replay — replay an arbitrary alert rule against history.
  • Bankroll + Kelly — persist your bankroll, get sized stake suggestions per signal.

Automation

  • Rule-based alerts — structured filters by sport, market, divergence, time, channel routing, quiet hours.
  • Bearer API tokens — programmatic access via Authorization: Bearer <token>.
  • Signed webhooks — HMAC-SHA256 signatures, replay protection.
  • PWA + Web Push — install on phone, real-time push when signals fire.

AI

  • Signal explanations — Claude generates plain-English "why is this mispriced" for any signal. Cached aggressively to bound API cost.

Real-time foundation

Latency

  • Polymarket WebSocket subscriber: prices update within ~2s of the venue (was 5 min poll).
  • Adaptive per-sport polling: 15s pre-game, 60s within 4h, 5 min today, 30 min idle.
  • Odds API quota-aware throttling so the free 500/mo tier survives multiple sports.
  • Circuit breaker on the PM WebSocket — won't hammer Polymarket if they go down.

Signal quality

  • De-vigged divergence (strips bookmaker overround before comparing).
  • Sharp-consensus gate (require Pinnacle/Circa/BetCRIS/Betfair Ex).
  • Liquidity gate (≥$1k Polymarket volume, spread ≤ 5pp).
  • Stale-data gate (rejects markets that have never traded or are flat for a week).

Refactor + observability

  • Main file sports_dashboard.py shrunk 44% by extracting embedded HTML into templates/.
  • Pinned requirements, split into requirements.txt + requirements-dev.txt.
  • Prometheus metrics at /metrics — 25+ metrics across poll loop, signals, alerts, quota, fills, push.
  • Liveness + readiness probes at /healthz and /readyz.
  • 289 tests covering matching, signal-quality gates, track record, alerts, tokens, webhooks, push, leaderboard.
