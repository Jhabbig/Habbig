Trust + Proof

Track record Public

CLV, simulated P&L, and calibration of every flagged signal over the last 30/90/180/365 days. Tunable threshold + stake.
/track-record · /api/track-record/{clv,pnl,calibration}

CLV leaderboard Public

Opt-in roster of sharp users ranked by mean closing line value. Minimum 10 resolved trades to qualify.
/leaderboard · /api/leaderboard/clv

Cross-venue signals

Live dashboard

Book × Polymarket × Kalshi divergence feed for the active sport, with sharp-consensus, liquidity, and staleness gates applied. Real-time WS push for live price moves.
/ · /api/data · WebSocket /ws

Player props

Bookmaker player props (DraftKings, FanDuel, BetMGM) joined against Kalshi props on (player, stat, line). EV signals at divergence ≥ 5pp.
/player-props · /api/player-props/cross-venue

Cross-book arbitrage

Low-hold (negative-vig) and middling opportunities across configured books. Pure book-vs-book — no Polymarket needed.
/cross-book-arbitrage · /api/cross-book-arbitrage

Differentiation moats

Smart money New

Top-50 Polymarket wallets' positions overlaid on the active sport. "23 whales hold this position with $X exposure" — nobody else surfaces this for sports markets.
/smart-money · /api/smart-money

Polymarket fills tape Live

Real-time feed of large Polymarket fills (≥$1k by default). See sharp money landing before the price catches up.
/poly-fills · /api/poly-fills

Steam moves

Sharp-book line moves ≥2pp within 30 min. Tunable threshold and window. Sorted by absolute swing.
/steam-moves · /api/steam-moves

Your edge

Bet tracker

Log placed bets, track P&L and per-trade CLV, break down by sport/book/market type, export CSV.
/trades · /api/trades/* · /api/trades/csv

Backtest replay

Replay an arbitrary alert rule against resolved historical signals. Tune filters; see hypothetical P&L, win rate, Sharpe, max drawdown.
/backtest · /api/backtest/replay

Bankroll + Kelly

Persist your bankroll, configure Kelly fraction and per-bet cap. Get sized stake suggestions per signal with drawdown alerts.
/api/bankroll · /api/bankroll/suggest-stake

Pro automation

Rule-based alerts

Build structured alert rules (sport, market type, divergence floor, volume, time-to-event, quality gates, quiet hours, channel routing).
/api/alert-rules (GET, POST, PATCH, DELETE)

API tokens

Bearer tokens for programmatic access. Authorization: Bearer <token>. Tokens are SHA-256 hashed at rest; plaintext shown once.
/api/auth/tokens (GET, POST, DELETE)

Signed webhooks

HMAC-SHA256 signatures on every webhook payload via X-Sharpe-Signature. Verify with constant-time compare; replay protection via X-Sharpe-Timestamp.
/api/webhooks/signing-key · /api/webhooks/test

Browser push (PWA)

Install the dashboard as a PWA, subscribe to Web Push for real-time signals on phone or desktop even when the tab is closed.
/api/push/* · manifest.json · sw.js

AI

Signal explanation

Plain-English "why is this mispriced" for any signal, generated by Claude. Cached 30 min per signal identity to bound API cost.
POST /api/signals/explain

Operational

Prometheus metrics

25+ metrics including poll-loop duration, comparisons, signals, alert delivery, WS reconnects, Odds API quota, fill capture counts, push delivery.
/metrics (no auth — bind to private network)

Diagnostics

Recent fuzzy-match near-rejects (helps tune team aliases) and Odds API quota tracker (low-water mark, exhaustion count). Admin-only.
/api/diagnostics/{match-rejects,odds-quota}
