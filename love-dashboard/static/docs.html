Dashboard Methodology API

API reference

All endpoints are HTTP GET, return JSON, and respect Access-Control-Allow-Origin: *. No auth, no rate limits beyond what the upstream data sources impose. Public, read-only.

Custom weights: any endpoint that returns scored countries accepts ?w_connection=<0..1>&w_partnership=<0..1>&w_stability=<0..1>&w_activity=<0..1>. Weights renormalize to sum to 1; partial sets are merged with defaults.

Health & introspection

GET /api/health
Liveness probe. Returns {ok: true, ts: }.
curl https://love.narve.ai/api/health
GET /api/sources
Methodology constants and the live status of every data feed.
curl https://love.narve.ai/api/sources | jq

Index payloads

GET /api/summary
Single-page payload: global mean composite, subscore averages, coverage counts, top/bottom 10, weights, weights_customized flag.
Query params: w_* (custom weights)
curl 'https://love.narve.ai/api/summary?w_connection=0.6&w_partnership=0.2&w_stability=0.15&w_activity=0.05'
GET /api/index
Every ranked country with its subscores, raw indicators, and which subscores were used.
Query params: w_* (custom weights)
curl https://love.narve.ai/api/index
GET /api/countries
Ranked + unranked rows. Unranked rows include unranked_reason for honest data-gap reporting.
Query params: w_* (custom weights)
curl https://love.narve.ai/api/countries
GET /api/country/<iso3>
One country's record + peer_compare (income-tier mean and Δ for each subscore) + sensitivity block (default weights only).
Path: ISO3 country code. Query: w_*.
curl https://love.narve.ai/api/country/USA

Insights & sensitivity

GET /api/insights
All insight cards generated by the 11 rules. Each has kind (e.g. "outlier"), severity, confidence, title, body, and pointers to the raw values.
Query params: w_*
curl https://love.narve.ai/api/insights | jq '.[] | {kind, title}'
GET /api/sensitivity
Per-country rank ranges across 13 weight perturbations. Each country gets a stability label (high / medium / low). Includes most_unstable, most_stable, and stability_distribution.
curl https://love.narve.ai/api/sensitivity | jq '.stability_distribution'

History

GET /api/history/<iso3>
Country composite + subscore time series, oldest first.
Path: ISO3. Query: days (default 365, max 3650).
curl 'https://love.narve.ai/api/history/USA?days=730'
GET /api/history/global
Daily mean composite across ranked countries plus the sample size, oldest first.
Query: days (default 365, max 3650).
curl 'https://love.narve.ai/api/history/global?days=3650'

LLM narrative

GET /api/narrative/<iso3>
2–3 paragraph analyst note generated by Claude Haiku 4.5 from the country's snapshot, history, context indicators, and active insights. Cached per (iso3, UTC date) — at most one Claude call per country per day. Returns {text, model, generated_at, usage}. Returns 503 if ANTHROPIC_API_KEY is unset on the server.
curl https://love.narve.ai/api/narrative/USA
GET /api/narrative/<iso3>?stream=1
Same narrative as above but returned as Server-Sent Events so the UI can render tokens live. Emits {type:"delta",text:...} events as Claude writes them, terminating with one {type:"done", text, model, generated_at, usage}. Cache-aware: a same-day re-open emits the cached text in one delta + done immediately (no Claude call).
curl -N -H 'Accept: text/event-stream' https://love.narve.ai/api/narrative/USA
GET /api/narrative/compare/<iso_a>/<iso_b>
Comparative analyst note for two countries — same shape as the single-country narrative but framed as a contrast. Uses a dedicated COMPARE_SYSTEM_PROMPT with its own prompt-cache entry. Cached per (iso_a, iso_b, UTC date). Supports ?stream=1 for SSE. iso_a == iso_b returns 400.
curl -N -H 'Accept: text/event-stream' https://love.narve.ai/api/narrative/compare/USA/DEU?stream=1

Exports

GET /api/export/countries.csv
Current rankings as CSV — rank, ISO, name, tier, region, composite, stability label, all four subscores, raw indicators, all six context indicators. Respects the custom-weight query params (w_*) so the file matches the user's reweighted view.
curl -OJ 'https://love.narve.ai/api/export/countries.csv'

Share assets

Operator tools

Backfill history: a one-shot CLI that seeds data/snapshots.db with year-end snapshots from Eurostat & World Bank time series, going back to whatever --start-year you pass.
python3 backfill.py --start-year 2010
python3 backfill.py --start-year 2015 --dry-run
